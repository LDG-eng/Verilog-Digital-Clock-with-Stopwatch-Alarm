# Experiment Setup

This document describes the **simulation environment**, **testbench structure**, and **verification procedure** for the Verilog Digital Clock project with **Stopwatch** and **Alarm** functionalities.

---

## 📌 Simulation Environment

- **Simulation Tool:** ModelSim  
- **Directory Structure:**  
tb/
├── Top_module_tb.v # Top-level module testbench
├── alarm_clk_tb.v # Alarm & current time module testbench
├── clock_generator_tb.v # Clock generation module testbench
└── stopwatch_tb.v # Stopwatch module testbench

- **RTL Source Directory:** `rtl/`  
- **Waveform Output:** VCD files generated for viewing in ModelSim waveform viewer  

> All testbenches are **module-level**, focusing on functional verification of each block individually and integration at the top-level.

---

## ⚡ Constraints

Constraints were provided in the course and are applied in simulation to ensure correct timing behavior.  
- **Clock Inputs:**  
- Main clock: 5 KHz (`Clock_5K`)  
- Derived clocks: 1 second (`Clock_1Sec`) and 1 millisecond (`Clock_1MSec`) generated by `clock_gen` module  
- **Reset:** Active-high reset applied to all modules  
- **Control Signals:** LoadTime, LoadAlm, AlarmEnable, Start_S, Stop_S, Reset_S as specified in the RTL module specs  

> Note: Exact FPGA/ASIC synthesis constraints (timing, area) are `[확실하지 않음]`.

---

## 🛠️ Verification Procedure

1. **Module-Level Simulation:**  
 - Each module is verified using its dedicated testbench.  
 - Simulation scenarios include:  
   - **clock_generator_tb.v:** Verify 1Sec and 1MSec clock generation from 5 KHz input.  
   - **alarm_clk_tb.v:** Verify timekeeping, AM/PM transition, alarm trigger logic, and time/alarm load operations.  
   - **stopwatch_tb.v:** Verify start, stop, reset behavior at millisecond resolution.  

2. **Top-Level Simulation:**  
 - `Top_module_tb.v` integrates all modules and verifies:  
   - Correct mode switching (Clock ↔ Stopwatch)  
   - Combined operation of Alarm and Stopwatch  
   - Display output correctness (`Hours`, `Mins`, `Secs`, `MSecs`, `AM_PM`, `Alarm`, `SW_State`)  

3. **Waveform Analysis:**  
 - Simulation results are captured in VCD files.  
 - Key signals monitored: Clocks, counters, AM/PM, Alarm flag, Stopwatch outputs.  
 - Waveforms are stored in `../assets/waveforms/` and screenshots in `../assets/images/`.

---

> This setup ensures **functional correctness** of all RTL modules before synthesis, suitable for demonstration in portfolio and integration into FPGA/ASIC flows.
