# Experiment Setup

This document describes the **simulation environment**, **testbench structure**, and **verification procedure** for the Verilog Digital Clock project with **Stopwatch** and **Alarm** functionalities.

---

## 📌 Simulation Environment

- **Simulation Tool:** ModelSim
- **Directory Structure:**
  - 📂 **`tb/`**
    - 📜 `Top_module_tb.v` (Top-level module testbench)
    - 📜 `alarm_clk_tb.v` (Alarm & current time module testbench)
    - 📜 `clock_generator_tb.v` (Clock generation module testbench)
    - 📜 `stopwatch_tb.v` (Stopwatch module testbench)
- **RTL Source Directory:** `rtl/`
- Waveform Output: Simulation waveforms observed using ModelSim waveform viewer.

> All testbenches are **module-level**, focusing on functional verification of each block individually and integration at the top-level.

---

## ⚡ Simulation Constraints & Parameters

Constraints were applied in simulation to ensure accurate timing behavior matching the project specifications.

- **Simulation Timescale:** `` `timescale 10us/1us ``
- **Clock Inputs:**
  - Main clock: 5 KHz (`Clock_5K`)
  - Derived clocks: 1 second (`Clock_1Sec`) and 1 millisecond (`Clock_1MSec`) generated by the `clock_gen` module.
- **Reset:** Global active-high reset signal applied to initialize all modules safely.
- **Control Signals:** System state is driven by input stimuli (`LoadTime`, `LoadAlm`, `AlarmEnable`, `Start_S`, `Stop_S`, `Reset_S`) as defined in the RTL specifications.

> Note: Physical synthesis constraints (Area, Power) and static timing analysis were not the focus of this verification stage.

---

## 🛠️ Verification Procedure

### 1. Module-Level Simulation
Each module is verified using its dedicated testbench to isolate and confirm specific behaviors:
- **`clock_generator_tb.v`:** Verify 1Hz (1Sec) and 1KHz (1MSec) clock generation accuracy from the 5KHz input.
- **`alarm_clk_tb.v`:** Verify timekeeping increment logic (0-59 for seconds/minutes, 1-12 for hours), AM/PM transitions, alarm trigger matching, and time/alarm manual load operations.
- **`stopwatch_tb.v`:** Verify state machine transitions (start, pause, stop, reset) at millisecond resolution and ensure proper signal priority (e.g., `Reset_S` overrides `Stop_S`).

### 2. Top-Level Simulation
`Top_module_tb.v` integrates all sub-modules and verifies system-level behavior:
- Correct mode switching (Clock ↔ Stopwatch) using the `Control` signal without signal glitches.
- Concurrent, independent operation of the Alarm and Stopwatch in the background.
- Display output correctness (`Hours`, `Mins`, `Secs`, `MSecs`, `AM_PM`, `Alarm`, `SW_State`).

### 3. Waveform Analysis
- Simulation results are captured and analyzed to ensure all transitions meet the design specifications.
- **Key signals monitored:** Clocks, individual counters, AM/PM toggle, Alarm flag, and Stopwatch states.
- Output waveforms are stored in `../assets/waveforms/`, and screenshots are available in `../assets/`.

---

> This setup ensures the **functional correctness** and stability of all RTL modules prior to physical synthesis, confirming suitability for FPGA/ASIC implementation flows.
